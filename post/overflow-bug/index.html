A subtle bug Dat trigraph syntax 28 May 2016

Spot the bug:

int getAlphaComponent(int color) {
    // color is stored in ARGB format.
    return color >> 24;
}

Do you see it? It took me way to long.

In Java >> is signed/arithmetic shift not logical shift so 0xFF000000 >> 24 is 0xFFFFFFFF (-1) when you probably wanted 0x000000FF (255).

Like all the best bugs this one has a single character fix:

int getAlphaComponent(int color) {
    // color is stored in ARGB format.
    return color >>> 24;
}
